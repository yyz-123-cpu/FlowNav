# FlowNav 前端使用说明

## 概述

已创建完整的FlowNav前端原型，使用Vue.js + ECharts技术栈，实现路径规划、路径对比、时间敏感性分析等核心功能。

## 已创建的文件

1. **index.html** - 主页面，包含导航栏、控制面板、图表展示区
2. **style.css** - 样式文件，美化界面
3. **app.js** - Vue.js主应用，包含所有业务逻辑和ECharts集成
4. **mockData.js** - 模拟数据服务，提供校园节点、路径规划等模拟数据
5. **api.js** - API服务封装，支持模拟数据和真实API切换
6. **serve.py** - 简单HTTP服务器，用于测试前端
7. **bug_report.md** - 测试发现的bug报告

## 功能特性

### 1. 路径规划
- 选择起点和终点节点
- 调节用户偏好权重α (0.0-2.0)
- 设置规划时间
- 可视化展示路径和校园地图
- 显示路径详情（距离、成本、拥堵成本等）

### 2. 路径对比
- 对比不同α值的路径效果
- 显示距离、总成本、拥堵成本对比图表
- 表格形式展示详细数据

### 3. 时间敏感性分析
- 分析同一路线在不同时间点的表现
- 识别高峰期和平峰期的路径变化
- 图表展示时间与拥堵关系

### 4. 系统特性
- **混合模式API**: 支持模拟数据和真实API切换
- **响应式设计**: 适配不同屏幕尺寸
- **实时图表**: 使用ECharts进行数据可视化
- **交互式界面**: 友好的用户交互体验

## 快速开始

### 使用Python HTTP服务器

```bash
# 启动测试服务器
python serve.py

# 或者使用Python内置模块
python -m http.server 8080
```

然后在浏览器中访问：`http://localhost:8080`

## 使用指南

### 1. 首次使用
1. 启动HTTP服务器
2. 打开浏览器访问页面
3. 系统默认使用模拟数据模式
4. 尝试"路径规划"功能查看基本效果

### 2. 路径规划示例
1. 选择起点：第一教学楼 (TB1)
2. 选择终点：第一食堂 (CA1)
3. 调节α值观察路径变化
4. 点击"规划路径"按钮

### 3. 切换API模式
- 默认使用模拟数据
- 点击顶部横幅中的按钮可切换模式
- 如需连接真实后端API，需要启动后端服务并修改api.js中的配置

## 技术栈

- **前端框架**: Vue.js 3 (CDN引入)
- **图表库**: ECharts 5
- **样式框架**: Bootstrap 5
- **HTTP客户端**: Axios
- **图标库**: Font Awesome

## 已知限制

1. **模拟数据**: 当前使用模拟数据，路径规划结果基于预定义规则
2. **节点有限**: 模拟校园地图包含14个节点，覆盖主要建筑类型
3. **API集成**: 真实API模式需要后端服务支持（尚未实现）
4. **浏览器兼容**: 建议使用现代浏览器（Chrome, Firefox, Edge）

## 故障排除

### 常见问题

1. **图表不显示**
   - 检查网络连接，确保能访问CDN资源
   - 查看浏览器控制台错误信息
   - 尝试刷新页面

2. **服务器启动失败**
   - 检查端口8080是否被占用
   - 修改serve.py中的PORT变量
   - 确保Python已安装

3. **功能无响应**
   - 检查浏览器JavaScript控制台
   - 确认所有.js文件已正确加载
   - 尝试清除浏览器缓存

## 部署指南

FlowNav 前端可以通过多种方式部署到公网，实现随时随地访问：

### 快速部署 (ngrok + nginx)
推荐使用 ngrok + nginx 的轻量级部署方案，详细步骤请参考主 README.md 中的 [公网部署方案](../README.md#公网部署方案) 章节。

简要步骤：
```bash
# 1. 启动后端API服务器
python api_server.py --port 8000

# 2. 启动前端服务器
python serve.py --port 8080

# 3. 启动nginx反向代理
nginx.exe

# 4. 启动ngrok隧道
ngrok http 8080
```

### 静态站点托管
- **GitHub Pages**: 免费托管静态前端
- **Vercel**: 自动HTTPS + 全球CDN
- **Netlify**: 类似Vercel的替代方案

详细部署指南请参考主 README.md 中的 [公网部署方案](../README.md#公网部署方案) 章节。

### 配置说明
部署时需要更新 `api.js` 中的 `apiBaseUrl`，指向公网可访问的后端API地址。
